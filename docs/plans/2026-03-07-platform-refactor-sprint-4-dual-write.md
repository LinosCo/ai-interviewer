# Platform Refactor Sprint 4 — Dual Write

> For a lower-capability model: keep legacy behavior as the source of truth in this sprint. Canonical writes must be best-effort and must not break legacy flows.

**Goal**

Every new tip generated by legacy systems should also materialize into `ProjectTip`.

## Files To Modify

- `src/lib/insights/sync-engine.ts`
- `src/lib/visibility/website-analysis-engine.ts`
- `src/lib/visibility/brand-report-engine.ts`
- `src/lib/copilot/chat-tools.ts`

## Files To Import

- `src/lib/projects/project-tip.service.ts`

## Global Rules

1. Legacy write first, canonical write second.
2. Canonical write must run in `try/catch`.
3. If canonical write fails, log a namespaced warning and continue.
4. Do not change existing return payloads in this sprint.

## Task 1: Dual-write in CrossChannel Sync

In `src/lib/insights/sync-engine.ts`, locate `upsertInsight()` and the create/update path around the `crossChannelInsight.create` and `crossChannelInsight.update` calls.

After each successful create or update:

```ts
try {
  await ProjectTipService.materializeFromCrossChannelInsight(insight.id);
} catch (error) {
  console.warn('[project-tip-dual-write] cross-channel materialization failed', { insightId: insight.id, error });
}
```

Do not alter sync scoring or existing insight status behavior.

## Task 2: Dual-write in Website Analysis

In `src/lib/visibility/website-analysis-engine.ts`, after the final successful `websiteAnalysis.update` that writes `recommendations`, call:

```ts
try {
  await ProjectTipService.materializeFromWebsiteAnalysis(analysisId);
} catch (error) {
  console.warn('[project-tip-dual-write] website-analysis materialization failed', { analysisId, error });
}
```

Do not block analysis completion if canonical materialization fails.

## Task 3: Dual-write in Brand Report

In `src/lib/visibility/brand-report-engine.ts`, after the successful `brandReport.update` that writes `aiTips`, call:

```ts
try {
  await ProjectTipService.materializeFromBrandReport(reportId);
} catch (error) {
  console.warn('[project-tip-dual-write] brand-report materialization failed', { reportId, error });
}
```

Do not change report scoring or existing response shape.

## Task 4: Dual-write in Copilot Strategic Tip Creation

In `src/lib/copilot/chat-tools.ts`, keep the current `crossChannelInsight.create` path.

Immediately after the insight is created:

```ts
let canonicalTipId: string | null = null;
try {
  const result = await ProjectTipService.materializeFromCrossChannelInsight(insight.id);
  canonicalTipId = result.tipId;
} catch (error) {
  console.warn('[project-tip-dual-write] copilot tip materialization failed', { insightId: insight.id, error });
}
```

If convenient, include `canonicalTipId` in the tool response, but do not remove any legacy IDs or fields already returned.

## Task 5: Smoke Verification

Run:


- cross-channel sync

Verify that:

- all four dual-write hooks exist
- canonical write failures do not break legacy flows
- manual smoke test confirms new canonical rows appear
