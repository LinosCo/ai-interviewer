# AI Tips UX Unification ‚Äî Design Document

**Date:** 2026-03-14
**Branch:** stage
**Status:** Approved

## Problem

The AI Tips system has 3 parallel tip systems (VirtualInsight, CrossChannelInsight, ProjectTip) with different UIs, different CTAs, and different lifecycle management. Site Analysis recommendations appear as read-only in the AI Tips page. The page has 3 tabs (Listen, Tips, Strategy) creating fragmentation. The sidebar buries Site Analysis under Brand Monitor.

## Design

### A. Unified Tip Model

Everything becomes **ProjectTip**, materialized immediately at creation time.

**Categories (by origin):**
- `sito` ‚Äî from Site Analysis / Brand Report
- `ascolto` ‚Äî from Interviews / Chatbot
- `copilot` ‚Äî generated by Copilot
- `manuale` ‚Äî user-created

**CTAs on every tip (identical for all):**
- If routing integration is configured ‚Üí **"Applica"** (send to CMS/WordPress/etc)
- If no integration ‚Üí **"Approfondisci con Copilot"** (opens copilot chat preloaded with tip context)
- Always available: Archivia, Completa, Stella, Modifica

**Eliminated:**
- `site-analysis-insights.ts` (VirtualInsight layer)
- `isVirtual` logic in insights page
- "Read-only" footer on imported tips
- `AITipsSection.tsx` inside visibility (tips live in AI Tips page)

### B. Simplified Visible States

3 visible states + 1 terminal, derived automatically from backend fields:

| Visible State | Badge | When |
|---|---|---|
| **Nuovo** | Blue | Just created, no action taken |
| **In lavorazione** | Amber | Routing configured, OR draft generated, OR user clicked "Applica" |
| **Completato** | Green | Execution succeeded, OR manually marked by user |
| **Archiviato** | Grey | Dismissed by user |

**Auto-advancement:**
- Nuovo ‚Üí In lavorazione: automatic when routing is configured or draft is generated
- In lavorazione ‚Üí Completato: automatic on `markExecutionSuccess()`, or manual (button)
- Any ‚Üí Archiviato: always manual
- On `markExecutionFailure()`: stays "In lavorazione" with error indicator

Backend fields (`status`, `draftStatus`, `routingStatus`, `publishStatus`) remain in DB. UI uses a derived function `getVisibleStatus(tip)`.

### C. AI Tips Page Layout

**No tabs.** Single view with two areas:

#### Strategy Banner (top)
Compact banner showing strategy status. Button opens an **AI-powered strategy interview** ‚Äî a wizard that uses an AI bot to ask questions and extract objectives, target, positioning. Result saved as project strategy.

If not configured: CTA "Configura strategia" opens the wizard.
If configured: shows summary + "Modifica" button.

#### Tips List (body)
Linear/Notion-inspired minimal design:

- **Rows, not cards** ‚Äî dense, clean
- **One row = one tip**: title, priority dot (red/amber/blue), status badge aligned
- **Second line**: brief summary (max 1 line, truncated)
- **Primary CTA right-aligned**: contextual (Applica/Copilot)
- **`¬∑¬∑¬∑` menu** for secondary actions (Edit, Star, Complete, Archive)
- **Expand on click**: inline panel below the row with evidence, reasoning, alignment
- **Grouped by category**: lightweight header with count
- **Archived**: collapsed at bottom by default

**Filters:**
- Status: Tutti / Nuovi / In lavorazione / Completati / Archiviati
- Category: Tutte / Sito / Ascolto / Copilot / Manuale
- Toggle: Solo stellati

### D. Sidebar Navigation

**New order:**
```
App:
  Interviste AI
  Chatbot AI
  Brand Monitor        (flag: hasVisibilityTracker)
  Analisi Sito         (flag: hasSiteAnalysis) ‚Äî NEW, independent
  AI Tips              (flag: hasAiTips)
  Formazione
```

- `Analisi Sito` ‚Üí `/dashboard/site-analysis` (new route, reuses SiteAnalysisClient)
- Brand Monitor ‚Üí keeps prompt monitoring, brand tracking, SERP
- `hasSiteAnalysis` is an independent feature flag (separate from `hasVisibilityTracker`)

### E. Site Analysis Page

`/dashboard/site-analysis` maintains **all complete analysis results**:
- Score rings (overall, SEO, LLMO, GEO, SERP)
- Page audit details
- LLMO distribution

What changes: tips generated by analysis are materialized as ProjectTip and managed in AI Tips page, not displayed inline via `AITipsSection.tsx`.

### F. What Gets Removed

- `src/lib/insights/site-analysis-insights.ts` (VirtualInsight bridge layer)
- `isVirtual` logic throughout insights page
- Tab Listen, Tab Strategy (strategy becomes wizard in banner)
- `AITipsSection.tsx` from visibility context (tips in AI Tips page only)
- "Read-only" footer on imported tips
- `InsightCard.tsx` (replaced by minimal rows)
- Health Report section inline in insights page
